# Feature PRD: File-basesd Specifications

## Overview

Playback specifications are provided in the url path. This is a straightforward way to get started and use the tool. However, it becomes difficult to specify more sophisticated scenarios or to create scenarios that are shared across multiple projects. Allowing the developer to create named specifications, that can be loaded by a given name rather than the actual specification, would make it easier to re-use scenarios across projects. It would also provide for more flexibility in the specification framework. 

## Goals

- More expressive specification method
- Allow for more reliable scenario specifications in more than one location (usually across platforms for standard scenario testing) 
- Scenarios specifications that can be managed in version control systems
